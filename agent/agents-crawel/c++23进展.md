# C++ 标准进展 (截至 2026.06)

## ✅ C++23 — 已发布 (ISO/IEC 14882:2023, 2023年11月)

### 核心语言特性
- `std::print` / `std::println` — 类型安全的格式化输出，替代 `printf`/`cout`
- `std::expected<T, E>` — 标准化的预期类型（错误处理），替代 `std::optional`
- `std::mdspan` — 多维视图，支持科学计算和 HPC
- `std::stacktrace` — 首次标准化堆栈追踪能力
- `std::generator<T>` — 协程支持的懒序列生成器（类似 Python `yield`）
- `std::flat_map` / `std::flat_set` — 基于 `std::vector` 的有序容器（缓存友好）
- 范围适配器管道语法增强（`views::filter | views::transform`）
- 更强的 Modules 支持

### 编译器支持
| 编译器 | 版本 | C++23 支持 |
|--------|------|-----------|
| GCC | 13 / 14 | 完整 (-std=c++23) |
| Clang | 17 / 18 / 19 | 完整 (-std=c++23) |
| MSVC | 19.38+ (VS 2022 17.8+) | 完整 (/std:c++latest) |

## 🚀 C++26 — 预计 2026 年发布（草案收尾阶段）

### 已确认纳入的特性

#### 核心语言
- **Contracts（契约编程）** — `pre`/`post` 条件检查，编译期/运行时可选
- **Reflection（反射）** — `^` 操作符，编译期访问类型元信息
- **Pattern Matching（模式匹配）** — `inspect` 表达式，替代冗长的 `switch`/`visit`
- **`std::execution`（Senders/Receivers）** — 标准异步模型，统一协程/线程池/GPU
- **`std::simd`** — 可移植 SIMD 向量类型，替代编译器内建 intrinsic
- **`std::hive`** — 稳定删除且缓存友好的容器（替代 `std::list` 场景）
- **`std::submdspan`** — `std::mdspan` 的子视图操作
- **`std::inplace_vector`** — 栈上固定容量动态数组
- 改进的 `std::format`（编译时格式字符串检查）
- 未初始化变量默认零初始化（安全性增强）

#### 标准库
- `std::text_encoding` — 标准文本编码处理
- `std::debugging` — 调试辅助工具
- `std::linalg` — 线性代数基础（BLAS 子集）

### 编译器支持进度
| 编译器 | 版本 | C++26 状态 |
|--------|------|-----------|
| GCC | 15 / 16 (开发中) | 部分支持 (-std=c++26) |
| Clang | 20 / 21 | 部分支持 (-std=c++26) |
| MSVC | 19.42+ | 部分支持 (/std:c++latest) |

### 关键时间节点
- 🏁 **Feature Freeze**: 2025 Q1（已完成）
- 📋 **委员会投票**: 2026 Q2（进行中）
- 📦 **正式发布**: **预计 2026 年下半年**

## 🛠️ 工具链与生态进展 (2026)

### 构建系统
- **CMake 4.0+** — 原生 C++20 Modules 支持 (`CXX_SCAN_FOR_MODULES`)
- **Ninja 1.12+** — 模块依赖扫描优化
- **vcpkg** / **Conan 2** — C++23/26 库支持持续完善

### 包管理
- `fmt` 11.x — 实验性 C++26 `std::format` 兼容
- `range-v3` — C++20/23 range 参考实现
- `boost` 1.87+ — C++23 部分特性支持

### AI/嵌入式 C++ 趋势
- **C++ for AI**: CUDA C++ 持续融合标准特性，SYCL 2020 生态扩展
- **嵌入式 C++**: MISRA C++:2023 发布，强化安全编码规范
- **安全编程**: NSA/CISA 推动 C++ 内存安全，`std::expected` + Contracts 成推荐范式
